Cleanup status

What is centralized now

  • Chart rendering goes through _build_chart_bytes in the Flask app.
  • Workspace switching and workspace save flows now route through shared helpers.
  • Parser heuristics for JSON, ZIP, headerless files, and workbook ranking are concentrated in the ingestion layer.
  • Prepare and visualize no longer keep live fallback bodies for workspace switching and workspace-save logic.
Still worth cleaning

Residual inline logic

  • Page-level chart rendering still lives inline in prepare.html and visualize.html.
  • The next structural cleanup is extracting those page scripts into separate JS modules once the UX stops shifting.
  • Some old branding assets remain on disk and can be pruned in the next asset cleanup pass if still unused.
